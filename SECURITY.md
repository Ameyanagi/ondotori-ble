# Security policy

## Supported versions

During alpha development, only the newest published release receives fixes.
The library must not be used as the sole component of a life-safety, alarm, or
regulatory-compliance system.

## Reporting a vulnerability

Use GitHub's private security-advisory reporting for the repository. Do not put
device serials, BLE identifiers, private captures, credentials, or restricted
T&D documents in a public issue.

Include the affected version, platform, impact, reproduction steps, and a
sanitized proof of concept. Acknowledgement is targeted within seven days; a
fix timeline depends on severity and hardware availability.
